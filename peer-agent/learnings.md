# peer-agent/learnings.md

Raw meta-observations from peer-agent exchanges. Only distilled form
graduates to global memory (`feedback_peer_*.md`). Domain facts about
the other agent's project DO NOT land here.

**Allowlist — what IS recorded:**
- Communication patterns ("when peer says X, reframe as Y")
- Debugging tactics that generalize across projects
- Failure modes ("asking for status without anchoring -> generic summary")
- Meta-loop insights (how my prompting shapes their output)

**Blocklist — what is NOT recorded:**
- Their code / file paths / architecture / infra
- Specific problem-scoped debugging steps
- Secrets, PII, LinkedIn profile URLs, company names
